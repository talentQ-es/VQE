Introducci√≥n al Modelo de Ising#

El Modelo de Ising es un modelo para entender los materiales magn√©ticos en t√©rminos de los spines de los √°tomos y los electrones que lo forman. Pdemos imaginar el s√≥lido magn√©tico como una red cristalina, y en cada uno de los nodos tenemos una flecha que apunta hacia arriba (\(\uparrow\)) o hacia abajo (\(\downarrow\)). Si dos flechas contiguas apuntan en la misma direcci√≥n (\(\uparrow \uparrow\) o \(\downarrow \downarrow\)), esta uni√≥n tiene una contribuci√≥n negativa a la energ√≠a del sistema: el sistema tiende a alinear los spines entre si. Por otro lado si las flechas apuntan en direcciones contrarias (\(\uparrow \downarrow\)), la energ√≠a de esa uni√≥n es positiva: al sistema le ‚Äúcuesta‚Äù mantener flechas en direcciones opuestas.

Existen dos factores externos que pueden condicionar el estado del sistema: el campo magn√©tico y la temperatura a los que lo sometemos. Por un lado, si introducimos un campo magn√©tico apuntando hacia arriba, los spines tienden a alinearse con el campo, por lo que todos querr√°n apuntar hacia arriba. Por otro lado, si calentamos mucho el sistema, las excitaciones t√©rmicas provocar√°n que los spines se agiten, cambiando de direcci√≥n y desorden√°ndose.

En el caso cu√°ntico, las fluctuaciones cu√°nticas del sistema provocan cambios en estos spines incluso a temperatura cero, permitiendo la existencia de transiciones de fase cu√°nticas.

Podemos expresar el Hamiltoniano del sistema como:

\[ \mathcal{H} = - J \sum_{\langle i,j \rangle} \sigma_i \sigma_j - h \sum_i \sigma_i \;, \]

donde \(J\) es la contribuci√≥n energ√©tica de la interacci√≥n de dos spines adyacentes (fij√©monos en que en el primer sumatorio solo se suma a primeros vecinos, \(\langle i,j \rangle\)) y \(h\) est√° relacionado con este campo magn√©tico externo. \(\sigma_i\) es la ‚Äúetiqueta‚Äù del spin y vale \(+1\) o \(-1\) seg√∫n apunte hacia arriba o hacia abajo.

Este problema puede complicarse dados diferentes factores microc√≥picos como la estructura cristalina o incluso tener interacciones m√°s all√° de los primeros vecinos, lo que requiere m√©todos aproximados para solucionarlo. No obstante, hay dos casos especiales para los que existe una soluci√≥n exacta y anal√≠tica del modelo: cuando el sistema tiene solo una dimensi√≥n o cuando el sistema tiene dos dimensiones y una estructura cuadrada.

El Modelo de Ising 1D#

Vamos a analizar el caso cu√°ntico de la cadena de Ising unidimensional. Precisamente por la naturaleza cu√°ntica del problema podemos utilizar las matrices de Pauli para caracterizar el Hamiltoniano, dando lugar a una nueva forma del operador [5]:

\[ \mathcal{H} = -J \sum_{i=1}^N X_i X_{i+1} - J g \sum_i Z_i \;, \]

siendo \(N\) la longitud de la cadena y, de nuevo, \(J\) est√° relacionado con la energ√≠a del sistema y \(g\) con la intensidad del campo magn√©tico a primeros vecinos. \(X_i\) y \(Z_i\) son operadores de Pauli como los que introduc√≠amos al principio de este documento, actuando sobre el spin en posici√≥n \(i\).

Este modelo se conoce como transverse field Ising Model. A veces se define \(\Gamma = J g\), donde \(\Gamma\) se conoce como campo transverso.

Nota (Notaci√≥n reducida)

En lo que a la descomposici√≥n de Pauli se refiere, los productos anteriores realmente son productos tensoriales de operadores para las part√≠culas \(i\) e \(i+1\), teniendo identidades en el resto de posiciones. Por ejemplo, para una cadena de tres spines, \(N=3\),

\[ \mathcal{H} = -J(X \otimes X \otimes I + I \otimes X \otimes X + X \otimes I \otimes X) - J g (Z \otimes I \otimes I + I \otimes Z \otimes I + I \otimes I \otimes Z) \;, \]

donde hemos tomado condiciones de contorno peri√≥dicas (\(\sigma_{N+1} \rightarrow \sigma_1\)).

PAULIS = {
    "I": np.eye(2, dtype=complex),
    "X": np.array([[0,1],[1,0]], dtype=complex),
    "Y": np.array([[0,-1j],[1j,0]], dtype=complex),
    "Z": np.array([[1,0],[0,-1]], dtype=complex)
}

Vamos a redefinir nuestras constantes del problema \(J\) y \(g\) de forma que podamos expresarlas como magnitudes relativas, escribiendo nuestro operador como

\[ \mathcal{H} = \sum_{i=1}^N X_i X_{i+1} + \gamma \sum_i Z_i \;. \]

Ahora los signos (\(-\)) han pasado a ser positivos y solo tenemos una constante de acoplo, \(\gamma\). Esto implica que, cuando \(\gamma\) es positivo estamos en el caso ferromagn√©tico y cuando es negativo en el antiferromagn√©tico.

La funci√≥n que implementa esto es la siguiente:


def ising_chain_ham_qibo(n, gam):
    
    # Esta funci√≥n nos devuelve el Hamiltoniano en t√©rminos utilizables por los algoritmos de Qibo
    # n = number of spin positions
    # gam = transverse field parameter

    from qibo.symbols import X, Z

    sym_X = sum(X(i) * X(i+1) for i in range(n-1)) + (X(0) * X(n-1))
    sym_Z = sum(Z(i) for i in range(n))
    sym_H = sym_X + gam * sym_Z

    H = hamiltonians.SymbolicHamiltonian(sym_H)
    
    return H, H.matrix


def ising_chain_ham(n, gam):
    
    # Esta funci√≥n nos devuelve el Hamiltoniano en t√©rminos utilizables por los algoritmos de Qiskit
    # n = number of spin positions
    # gam = transverse field parameter
    
    for i in range(n):
        vecX = [I] * n
        vecZ = [I] * n
        vecX[i] = X
        vecZ[i] = Z
        
        if i == n - 1:
            vecX[0] = X
        else:
            vecX[i+1] = X

        auxX = vecX[0]
        auxZ = vecZ[0]
        
        for a in vecX[1:n]:
            auxX = auxX ^ a
        for b in vecZ[1:n]:
            auxZ = auxZ ^ b
            
        if i == 0:
            H = (auxX) + (gam * auxZ)
        else:
            H = H + (auxX) + (gam * auxZ)

    return H


def ising_chain_matrix(n, gam):
    
    # Esta funci√≥n devuelve el Hamiltoniano como un array (lista de listas)
      
    for i in range(n):
        HX = [PAULIS["I"]] * n
        HZ = [PAULIS["I"]] * n
        HX[i] = PAULIS["X"]
        HZ[i] = PAULIS["Z"]
        
        if i == n - 1:
            HX[0] = PAULIS["X"]
        else:
            HX[i+1] = PAULIS["X"]
            
        aux = reduce(np.kron, HZ)
        aux = [val * gam for val in aux]
        
        if i == 0:
            H = reduce(np.kron, HX) + aux
        else:            
            H = H + reduce(np.kron, HX) + aux
        
    return H

Vamos a resolver una cadena de Ising de \(N = 4\) spines. Entonces prepararemos el Hamiltoniano y despu√©s le haremos una serie de transformaciones que nos permiten diagonalizarlo.

Empezamos por escribir nuestro operador. Como tenemos \(N=4\) spines, el operador ser√°

\[ \mathcal{H} = X_1X_2I_3I_4 + I_1X_2X_3I_4 + I_1I_2X_3X_4 + X_1I_2I_3X_4 + \gamma (Z_1I_2I_3I_4 + I_1Z_2I_3I_4 + I_1I_2Z_3I_4 + I_1I_2I_3Z_4) \]

Tomemos un valor de \(\gamma\) peque√±o, por ejemplo, \(\gamma = 0.3\).

Nota (Eficiencia en la optimizaci√≥n)

El algoritmo de Powell que estamos empleando es considerablemente m√°s eficiente que el resto de algoritmos que hemos probado para este problema espec√≠ficamente. No obstante, su funcionamiento da lugar a unas gr√°ficas de convergencia que no son particularmente ilustrativas. Si se quiere ver una gr√°fica en la que realmente se aprecie la convergencia del algoritmo, se puede probar a utilizar el algoritmo COBYLA.
